# Sketch direction (locked 2026-06-30, Option A)

**Workspace Sketch** = the real feature. Takes the current conversation and synthesizes it into the first visual design direction (concept / flow / UI / architecture / product). That visual becomes the design basis Atlas builds from.

**Homepage Sketch** = leave alone for now. Do NOT remove. Do NOT redesign with category pickers ("Product / Interface / Workflow…") — that's exactly the upfront-classification friction we've been removing.

**Long-term direction (not now):** Sketch becomes conversation-driven. User taps → Atlas asks "Show me what you're imagining." → Atlas infers the appropriate sketch type from intent. No categories surfaced to the user.

**Framing rule:** Sketch is not "draw something." It is "turn this conversation into a visual design direction." Copy and affordances should eventually reflect that, but only after the Workspace synthesis capability matures.

**Do not propose:** removing homepage Sketch, adding a category sheet on tap, renaming Sketch to "Visualize this project" yet, or any other reshuffle until the Workspace version is mature.
