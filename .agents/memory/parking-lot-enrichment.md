## Rule
Parking Lot uses two enrichment tiers:
- **Lite** (on park): `{ atlasCategory, complexity, whyItMatters, _level: "lite" }` — max_tokens 200, fast, cheap
- **Full** (on panel open): `{ ...lite, options, revisitWhen, whatItMeans, whyItComesUp, _level: "full" }` — generated by `deepEnrichParkedEntry()` when context endpoint is called

The `_level` field is the discriminator. The context endpoint (`POST /entries/:id/context`) checks it: full → return immediately, lite → upgrade in-place.

**Why:** Users may park many items quickly. Generating full analysis on every park wastes tokens on items that never get opened.

## ParkSheet
`ParkSheet.tsx` is the canonical park entry point for both Global Insight and Workspace. It:
- Uses `useCreateEntry` internally (self-contained)
- Accepts `projectId: number | null` — if null + multiple projects, shows a picker
- `onOpenFull` callback navigates to `/parking` (intentional management view)
- `onParked` callback for side effects (sound, refresh) in host components

## Resurfacing
Both surfaces inject parked items into Atlas's system prompt:
- **Nexus route** (Global Insight): already existed at lines ~1757-1765
- **Chat route** (Workspace): `parkedRows` added to parallel DB batch; DEFERRED ITEMS block injected after COMMITTED DECISIONS block

**How to apply:** When extending the park flow, keep enrichment in two tiers. Don't move deep enrichment to park time.
